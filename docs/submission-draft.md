### Project Description (250 words max)

**The Problem:** Enterprise teams managing Azure subscriptions lack a quick way to visualize their environment and get actionable security/cost insights without juggling multiple portals, CLI commands, and manual documentation.

**The Solution:** Azure Ops Dashboard connects to Azure Resource Graph, inventories every resource, and produces:

Reports are generated by the **GitHub Copilot SDK**, which streams AI analysis enriched with Microsoft Learn documentation references. Users can customize report templates (section ON/OFF, custom instructions) and switch between Japanese and English at runtime.

### Technical Highlights

1. **Copilot SDK streaming pipeline**: The app spawns a background thread that calls the Copilot SDK asynchronously. AI output is streamed token-by-token into the tkinter UI via atomic buffer swaps (replacing `list.clear()` with attribute rebinding to avoid race conditions between the writer thread and the 100ms UI poll timer).

2. **Microsoft Learn enrichment**: Before generating reports, the app queries Microsoft Learn Search API and a static reference map to inject grounded documentation URLs into the AI prompt â€” reducing hallucination and providing actionable links.

3. **Deterministic diagram layout**: The Draw.io generator uses a priority-ordered type list (LAYOUT_ORDER) to place resources in a consistent Region â†’ VNet â†’ Subnet hierarchy with Azure-official SVG icons. No random positioning â€” same input always produces the same diagram.

4. **Template system with user override**: Report templates (JSON) are bundled but can be overridden in `%APPDATA%\AzureOpsDashboard\templates\` without rebuilding the .exe. This "bundled + user override" pattern was a key design learning.

5. **Diff reports**: Each report run automatically finds the previous report and generates a structured diff, enabling environment drift tracking over time.

### Challenges & Learnings

- **Thread safety in tkinter**: Discovered that `StringVar.get()` from a background thread is unsafe even under GIL. Solution: capture all GUI values in the UI thread before spawning workers.
- **Copilot SDK timeout handling**: `asyncio.run_coroutine_threadsafe` timeouts leaked running coroutines. Added explicit `future.cancel()` on timeout.
- **PyInstaller resource paths**: `Path(__file__).parent` breaks in frozen executables. Created `app_paths.py` to abstract `sys._MEIPASS` fallback.
- **High-frequency `root.after()` calls**: Streaming AI output via per-token `after(0, ...)` caused UI freezes. Batched to 100ms polling with atomic buffer swap.
